# ProProgress - 智能任务提醒器

## 文件概述

`word_todo.html` 是一个单页面的任务管理Web应用，采用纯HTML/CSS/JavaScript开发，无需后端服务，使用浏览器本地存储保存数据。专为移动端和安卓APK优化，提供流畅的触摸交互体验。

## 技术栈

- **前端框架**: 纯HTML5 + CSS3 + Vanilla JavaScript
- **图标库**: Lucide Icons (通过CDN引入)
- **数据存储**: localStorage
- **通知API**: Browser Notification API
- **拖拽API**: HTML5 Drag and Drop API + Touch Events

## 功能特性

### 1. 任务管理
- **创建任务**: 支持任务名称、内容记录、子任务拆解
- **编辑任务**: 点击编辑图标修改任务名称、描述和子任务
- **删除任务**: 点击任务卡片右上角的X图标删除
- **子任务管理**: 每个任务可包含多个子任务，支持勾选完成
- **任务描述**: 支持为每个任务添加详细说明和记录（最多显示2行）

### 2. 拖拽排序
- **拖拽手柄**: 每个任务左侧有拖拽手柄图标
- **鼠标拖拽**: 支持桌面端鼠标拖拽排序
- **触摸拖拽**: 支持移动端触摸拖拽排序（垂直滑动触发）
- **视觉反馈**: 拖拽时显示半透明效果和虚线边框
- **自动保存**: 拖拽完成后自动保存顺序

### 3. 快捷操作
- **一键完成**: 点击任务名称行的完成图标，一键完成所有子任务
- **任务恢复**: 已完成任务可点击恢复图标重新激活
- **完成状态**: 已完成任务显示绿色"已完成"徽章和绿色进度条
- **批量操作**: 支持批量选择、完成和删除多个任务

### 4. 进度追踪
- **子任务进度**: 自动计算每个任务的完成百分比
- **总进度仪表盘**: 顶部显示所有任务的平均完成进度
- **实时更新**: 进度条和百分比实时更新
- **颜色区分**: 未完成任务显示紫色，已完成任务显示绿色

### 5. 智能提醒
- **浏览器通知**: 支持任务停滞提醒（30分钟未更新）
- **通知权限**: 点击顶部铃铛图标申请通知权限
- **定时检查**: 每分钟检查一次任务更新状态

### 6. 用户体验
- **扁平化设计**: 简洁现代的卡片式布局
- **流畅动画**: 所有交互都有平滑的过渡动画
- **触觉反馈**: 支持移动设备震动反馈
- **触摸优化**: 防止误触文本选择和长按菜单
- **响应式设计**: 完美适配移动端和桌面端
- **安全区域适配**: 支持刘海屏等特殊屏幕

### 7. 任务优先级
- **优先级设置**: 支持高、中、低三个优先级
- **颜色标识**: 不同优先级显示不同颜色（红色-高、黄色-中、蓝色-低）
- **优先级徽章**: 任务卡片显示优先级徽章
- **创建时设置**: 创建任务时可选择优先级
- **编辑时修改**: 编辑任务时可修改优先级

### 8. 任务分类/标签
- **标签添加**: 创建/编辑任务时可添加多个标签
- **标签显示**: 任务卡片显示所有标签
- **标签筛选**: 点击标签筛选器快速筛选任务
- **标签管理**: 自动收集所有标签，动态生成筛选器
- **标签样式**: 标签采用圆角设计，颜色区分

### 9. 深色模式
- **主题切换**: 点击顶部主题图标切换深色/浅色模式
- **自动保存**: 主题偏好自动保存到本地
- **全局适配**: 所有界面元素完美适配深色模式
- **视觉优化**: 深色模式下优化颜色对比度
- **图标切换**: 主题图标随模式切换（月亮/太阳）

### 10. 手势操作
- **左滑删除**: 向左滑动任务卡片显示删除按钮
- **右滑编辑**: 向右滑动任务卡片显示编辑按钮
- **双击完成**: 双击任务卡片快速完成/恢复任务
- **触摸拖拽**: 垂直滑动触发拖拽排序
- **视觉反馈**: 滑动时显示对应操作按钮

### 11. 数据备份与恢复
- **数据导出**: 支持导出所有任务数据为JSON文件
- **数据导入**: 支持从JSON文件恢复任务数据
- **版本信息**: 备份文件包含版本号和导出日期
- **主题备份**: 备份文件包含主题偏好设置
- **一键操作**: 简单的导出/导入操作界面

### JavaScript功能模块

#### 数据管理
```javascript
let tasks = JSON.parse(localStorage.getItem('pro_tasks_v3')) || [];
let editingIndex = -1;
let batchMode = false;
let selectedTasks = new Set();
let draggedItem = null;
let draggedIndex = null;
```
- 使用localStorage持久化存储
- 数据结构: `[{name, note, subtasks, updatedAt, isOpen}]`
- 支持编辑状态、批量模式、拖拽状态管理

#### 核心函数
- `init()`: 初始化应用，渲染图标，启动定时器
- `render()`: 渲染任务列表，更新进度，保存数据，初始化拖拽
- `calculateProgress(task)`: 计算单个任务的完成百分比
- `updateOverall()`: 更新总进度仪表盘

#### 交互函数
- `openAddModal()` / `closeAddModal()`: 打开/关闭添加任务弹窗
- `openEditModal(index)` / `closeEditModal()`: 打开/关闭编辑任务弹窗
- `handleCreate()`: 创建新任务
- `handleEdit()`: 保存编辑的任务
- `toggleSubtasks(index)`: 展开/收起子任务列表
- `toggleSubItem(tIdx, sIdx)`: 切换子任务完成状态
- `completeTask(index)`: 一键完成任务
- `uncompleteTask(index)`: 恢复未完成状态
- `deleteTask(index)`: 删除任务

#### 拖拽排序函数
- `initDragAndDrop()`: 初始化拖拽事件监听器
- `handleDragStart()`: 处理拖拽开始（鼠标）
- `handleDragEnd()`: 处理拖拽结束（鼠标）
- `handleDragOver()`: 处理拖拽经过
- `handleDragEnter()` / `handleDragLeave()`: 处理进入/离开目标区域
- `handleDrop()`: 处理放置操作
- `handleTouchStart()`: 处理触摸开始
- `handleTouchMove()`: 处理触摸移动（检测拖拽）
- `handleTouchEnd()`: 处理触摸结束
- `moveTaskTo(fromIndex, toIndex)`: 移动任务到新位置

#### 批量操作函数
- `toggleBatchMode()`: 切换批量操作模式
- `toggleSelect(index)`: 切换任务选择状态
- `batchDelete()`: 批量删除选中的任务
- `batchComplete()`: 批量完成选中的任务

#### 工具函数
- `formatTime(ms)`: 格式化时间显示
- `vibrate(p)`: 设备震动反馈
- `showToast(txt)`: 显示消息提示
- `toggleBell()`: 申请通知权限
- `checkReminders()`: 检查并发送提醒通知

## 使用说明

### 基本操作
1. **添加任务**: 点击底部输入框或"+"按钮打开添加弹窗
2. **填写信息**: 输入任务名称、描述、标签、优先级和子任务（用逗号或换行分隔）
3. **创建任务**: 点击"创建任务"按钮
4. **查看详情**: 点击任务卡片展开子任务列表
5. **完成子任务**: 点击子任务前的复选框
6. **编辑任务**: 点击任务卡片右上角的编辑图标
7. **删除任务**: 点击任务卡片右上角的X图标

### 优先级管理
1. **设置优先级**: 创建/编辑任务时选择优先级（高/中/低）
2. **优先级显示**: 任务卡片显示优先级徽章和颜色
3. **优先级颜色**: 高优先级-红色，中优先级-黄色，低优先级-蓝色
4. **修改优先级**: 编辑任务时可修改优先级

### 标签管理
1. **添加标签**: 创建/编辑任务时输入标签（用逗号分隔）
2. **查看标签**: 任务卡片显示所有标签
3. **筛选任务**: 点击顶部标签筛选器快速筛选任务
4. **全部任务**: 点击"全部"按钮显示所有任务
5. **标签样式**: 标签自动收集并显示在筛选器中

### 深色模式
1. **切换主题**: 点击顶部导航栏的主题图标（月亮/太阳）
2. **自动保存**: 主题偏好自动保存
3. **全局适配**: 所有界面元素完美适配深色模式
4. **视觉优化**: 深色模式下优化颜色对比度

### 手势操作
1. **左滑删除**: 向左滑动任务卡片，显示删除按钮
2. **右滑编辑**: 向右滑动任务卡片，显示编辑按钮
3. **双击完成**: 双击任务卡片快速完成/恢复任务
4. **触摸拖拽**: 按住任务卡片垂直滑动触发拖拽排序
5. **视觉反馈**: 滑动时显示对应操作按钮

### 数据备份与恢复
1. **导出数据**: 点击底部操作栏的"导出数据"按钮
2. **保存文件**: 浏览器自动下载JSON备份文件
3. **导入数据**: 点击底部操作栏的"导入数据"按钮
4. **选择文件**: 选择之前导出的JSON文件
5. **恢复数据**: 确认后恢复任务数据和主题设置

### 拖拽排序
1. **鼠标拖拽**: 按住任务左侧的拖拽手柄图标，拖动到目标位置
2. **触摸拖拽**: 在移动端，按住任务卡片并垂直滑动触发拖拽
3. **放置**: 释放鼠标或手指，任务会自动移动到新位置
4. **视觉反馈**: 拖拽时目标位置会显示虚线边框

### 快捷操作
1. **一键完成**: 点击任务名称行的绿色完成图标（✓）
2. **恢复任务**: 点击已完成任务的恢复图标（↺）
3. **查看进度**: 进度条和百分比实时显示完成度

### 批量操作
1. **进入批量模式**: 点击右上角"批量操作"按钮
2. **选择任务**: 点击任务卡片左侧的复选框选择多个任务
3. **批量完成**: 点击底部操作栏的"完成"按钮
4. **批量删除**: 点击底部操作栏的"删除"按钮
5. **退出批量模式**: 点击"取消"按钮

### 开启提醒
1. 点击顶部导航栏的铃铛图标
2. 在浏览器弹窗中选择"允许"
3. 应用会每分钟检查任务更新状态
4. 超过30分钟未更新的任务会收到通知提醒

## 设计特点

### 视觉设计
- **扁平化风格**: 简洁现代的卡片设计，减少视觉干扰
- **渐变背景**: 仪表盘使用紫色渐变
- **卡片式布局**: 任务以卡片形式展示
- **圆角设计**: 大圆角营造友好感
- **柔和阴影**: 轻微阴影增加层次感
- **紧凑布局**: 减少间距，提高信息密度

### 交互设计
- **即时反馈**: 所有操作都有视觉或触觉反馈
- **平滑过渡**: 展开/收起动画流畅自然
- **防误触**: 删除操作需要确认
- **空状态**: 无任务时显示友好提示
- **拖拽体验**: 流畅的拖拽排序，支持触摸和鼠标
- **触摸优化**: 防止文本选择，优化触摸响应

### 移动端优化
- **触摸友好**: 所有按钮尺寸适合触摸操作（最小44x44px）
- **防误触**: 禁用文本选择和长按菜单
- **触摸反馈**: 按钮按下时有缩放效果
- **震动反馈**: 关键操作提供设备震动
- **触摸拖拽**: 支持触摸屏拖拽排序
- **安全区域**: 适配刘海屏等特殊屏幕

### 性能优化
- **本地存储**: 数据保存在浏览器，无需网络请求
- **事件委托**: 减少事件监听器数量
- **按需渲染**: 只在数据变化时重新渲染
- **图标懒加载**: 使用CDN按需加载图标
- **拖拽优化**: 使用原生拖拽API，性能优异
